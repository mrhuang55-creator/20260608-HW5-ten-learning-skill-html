01

線性迴歸與殘差分析圖 (Residuals Plot)

線性迴歸（Linear Regression）旨在尋找自變量 $X$ 與連續目標值 $Y$ 之間的最佳線性組合。在經典統計學中，我們使用最小二乘法（Ordinary Least Squares, OLS）進行參數估計。

給定設計矩陣 $X \in \mathbb{R}^{N \times (D+1)}$（含偏置項列）與真實標籤 $y \in \mathbb{R}^N$，優化目標函數（殘差平方和）為：

$$L(w) = \frac{1}{2} \|y - Xw\|^2_2 = \frac{1}{2} (y^T y - 2 w^T X^T y + w^T X^T X w)$$

對 $w$ 進行矩陣求導並令其為零，可解得全局唯一解析解：

$$w^* = (X^T X)^{-1} X^T y$$

殘差診斷意義：評估迴歸模型好壞，不能僅看 $R^2$。殘差（$e_i = y_i - \hat{y}_i$）必須符合同方差假設 (Homoscedasticity)，即殘差必須隨機且均勻地分布在 $Y=0$ 兩側。若出現漏斗狀或拋物線形，則表示存在異質變異數或遺漏了高階非線性特徵。

線性擬合關係線

殘差診斷分佈（理想同方差）

sklearn_regression.py 實作代碼 
import matplotlib.pyplot as plt
from sklearn.linear_model import LinearRegression

# 建立模型並擬合
model = LinearRegression()
model.fit(X_train, y_train)

# 計算預測值與殘差
y_pred = model.predict(X_test)
residuals = y_test - y_pred

# 繪製殘差圖
plt.scatter(y_pred, residuals, alpha=0.5, color='indigo')
plt.axhline(y=0, color='red', linestyle='--')
plt.xlabel('Predicted Values')
plt.ylabel('Residuals')
plt.show()

思考與習題：

如果殘差圖呈現明顯的「向右開口漏斗狀」，這在統計學上稱為什麼？應該使用何種數學轉換（如對數轉換）來修正？

02

邏輯斯迴歸與 ROC 曲線 (ROC Curve & AUC)

邏輯斯迴歸（Logistic Regression）本質上是二分類模型。它通過引入**對數幾率（Log-Odds / Logit）**將預測空間壓縮至機率區間 $[0, 1]$：

$$\ln\left(\frac{p}{1-p}\right) = w^T x \implies p = \sigma(w^T x) = \frac{1}{1 + e^{-w^T x}}$$

利用**極大似然估計 (MLE)**，其對應的優化目標為最小化二元交叉熵損失（Binary Cross-Entropy Loss）：

$$J(w) = -\frac{1}{N} \sum_{i=1}^{N} \left[ y_i \ln(p_i) + (1-y_i) \ln(1-p_i) \right]$$

ROC 曲線與 AUC：ROC 曲線的橫軸為偽陽性率（$\text{FPR} = \frac{\text{FP}}{\text{FP+TN}}$），縱軸為真陽性率（$\text{TPR} = \frac{\text{TP}}{\text{TP+FN}}$）。我們藉由在 $[0,1]$ 區間內平移分類閾值，來記錄對應的點位。曲線下面積（AUC）越接近 1.0，代表分類器區分正負樣本的能力越強。

ROC 曲線與對角猜測基準線 (ROC Curve & AUC)

sklearn_roc.py 實作代碼 
from sklearn.metrics import roc_curve, auc

# 獲取預測正類之機率值
y_scores = model.predict_proba(X_test)[:, 1]
fpr, tpr, thresholds = roc_curve(y_test, y_scores)
roc_auc = auc(fpr, tpr)

# 繪圖展示 (AUC 值越大越佳)
print(f"ROC Area Under Curve (AUC): {roc_auc:.4f}")

思考與習題：

為什麼在不平衡數據集中（例如違約案件僅佔 0.1%），ROC 曲線有時無法客觀展現分類器對正類的預測表現？這時應該改用什麼曲線？

03

決策樹與空間劃分邊界 (Decision Boundary)

決策樹（Decision Tree）是一種非參數監督式學習算法。其核心思想是通過遞歸分割特徵空間，使得分割後的各子集純度（Purity）達到最高。

我們使用資訊熵（Information Entropy）或基尼不純度（Gini Impurity）來度量集合的不確定性。給定當前節點數據集 $D$：

$$\text{Entropy}(D) = -\sum_{k=1}^{K} p_k \log_2(p_k), \quad \text{Gini}(D) = 1 - \sum_{k=1}^{K} p_k^2$$

分裂特點與空間邊界：決策樹的分裂決策本質上是對單一維度特徵進行不平衡切分（即 $X_j > \theta$）。因此，決策樹在二維特徵平面上劃分出的決策邊界，必定是**正交階梯狀的直線**。這與曲線或超平面劃分模型具有本質上的幾何差異。

決策樹正交空間劃分邊界圖

dtree_visual.py 實作代碼 
from sklearn.tree import DecisionTreeClassifier
import numpy as np

# 限制樹的最大深度，防止過度擬合與邊界破碎
clf = DecisionTreeClassifier(max_depth=3, random_state=42)
clf.fit(X_train, y_train)

# 使用 meshgrid 計算密集的空間預測矩陣，用以繪製非線性邊界
xx, yy = np.meshgrid(np.arange(x_min, x_max, 0.01), np.arange(y_min, y_max, 0.01))
Z = clf.predict(np.c_[xx.ravel(), yy.ravel()]).reshape(xx.shape)

思考與習題：

若訓練集數據在空間中呈同心圓分佈（環形），單個決策樹在不進行任何座標軸轉換的情況下，其預測邊界會呈現何種狀態？為什麼其泛化能力會極差？

04

隨機森林與特徵重要性 (Feature Importance)

隨機森林（Random Forest）是經典的集成學習（Ensemble Learning）代表，通過結合多個決策樹的預測結果，顯著降低模型的方差（Variance）。它採用了 **Bootstrap Bagging** 與 **隨機特徵採樣** 的雙重隨機化技術。

特徵重要性 (MDI) 原理：對於森林中的任何一棵決策樹，在非葉子節點 $s$ 處，使用特徵 $X_j$ 分裂前與分裂後的基尼不純度減少量為 $\Delta \text{Gini}(s)$。全體樹在特徵 $X_j$ 分裂時所帶來的基尼不純度總體下降期望值，即為該特徵的重要度：

$$\text{Importance}(X_j) = \frac{1}{|M|} \sum_{T \in M} \sum_{s \in T, v(s)=X_j} p(s) \Delta \text{Gini}(s)$$

模型特徵重要度權重分析（Gini Importance）

rf_importance.py 實作代碼 
from sklearn.ensemble import RandomForestClassifier

# 初始化森林
rf = RandomForestClassifier(n_estimators=100, random_state=42)
rf.fit(X_train, y_train)

# 獲取重要度矩陣
importances = rf.feature_importances_
# 重要度權重加總必定為 1.0

思考與習題：

基於基尼不純度下降的特徵重要性，對於「高基數（Cardinarity）特徵」（如用戶 ID、身份證號等取值種類極多的變量）會產生什麼偏見？我們該如何解決？

05

支援向量機與最大間隔超平面 (SVM Margin)

支援向量機（Support Vector Machine, SVM）在特徵空間中尋找一個幾何間隔最大的超平面來劃分樣本。

對於線性可分數據，優化目標是在滿足約束條件 $y_i (w^T x_i + b) \ge 1$ 的前提下，最大化幾何邊界 $\frac{2}{\|w\|}$，等價於最小化：

$$\min_{w, b} \frac{1}{2} \|w\|^2_2$$

當面臨非線性特徵時，我們引入**核技巧（Kernel Trick）**將低維特徵映射到高維希爾伯特空間。徑向基核函數（RBF）公式為：

$$K(x_i, x_j) = \exp\left( -\gamma \|x_i - x_j\|^2 \right)$$

SVM 最大幾何邊界 (Margin) 與支援向量分佈

svm_kernel.py 實作代碼 
from sklearn.svm import SVC

# 使用高斯核函數 RBF
svm_clf = SVC(kernel='rbf', C=1.0, gamma='scale')
svm_clf.fit(X_train, y_train)

# 獲取哪些樣本點是決定模型邊界的關鍵支援向量
support_vectors = svm_clf.support_vectors_

思考與習題：

在 SVM 的損失函數中，正則化懲罰參數 $C$ 越小，模型的軟間隔（Soft Margin）會越寬還是越窄？這會使模型更容易過擬合還是欠擬合？

06

K-最近鄰與距離度量分析 (KNN Error Rate)

K-最近鄰（K-Nearest Neighbors, KNN）是一種基於實例的惰性學習（Lazy Learning）演算法。預測目標點的類別時，完全依賴特徵空間中與其最近的 $K$ 個鄰近樣本。

距離度量標準：歐氏距離（$L_2$ Norm）與曼哈頓距離（$L_1$ Norm）：

$$d_{\text{Euclidean}}(x, y) = \sqrt{\sum_{d=1}^{D}(x_d - y_d)^2}, \quad d_{\text{Manhattan}}(x, y) = \sum_{d=1}^{D} |x_d - y_d|$$

K 值的敏感度（偏差-方差權衡）：當 $K=1$ 時，模型完全契合最近的單個噪聲樣本點，決策邊界破碎，此時模型具有**低偏差、高方差**（極易過擬合）；當 $K$ 極大時，決策空間由全體大數據群體主導，決策邊界過於平滑，此時模型具有**高偏差、低方差**（欠擬合）。

K 鄰近值選擇與測試集錯誤率曲線 (KNN Bias-Variance Curve)

knn_test.py 實作代碼 
from sklearn.neighbors import KNeighborsClassifier

# 通常選擇奇數作為 K 值，防止二分類中出現平票情況
knn = KNeighborsClassifier(n_neighbors=5, metric='minkowski', p=2)
knn.fit(X_train, y_train)

思考與習題：

什麼是維度災難（Curse of Dimensionality）？在高維空間中，KNN 的歐氏距離計算為什麼會面臨失效？

07

K-Means 與肘部收斂法 (Elbow Method)

K-Means 是一種非監督式學習（Unsupervised Learning）聚類演算法。其優化目標是尋找 $K$ 個群集質心 $\mu$，使得各樣本點到所屬質心的**群集內誤差平方和（Within-Cluster Sum of Squares, WCSS）**降到最低：

$$\text{WCSS} = \sum_{j=1}^{K} \sum_{x_i \in C_j} \|x_i - \mu_j\|^2$$

肘部法則 (Elbow Method) 原理：隨着群集數量 $K$ 的增加，WCSS 會自然且必然地下降。然而，當 $K$ 小於真實群集數時，$K$ 的微小增加會大幅改善聚類的緊湊度（WCSS 急劇下墜）；一旦 $K$ 超過真實群集數，優化邊際效應遞減，WCSS 的下降斜率會突然變緩。此時曲線上的折折點（狀似手肘）即為最佳的聚類數 $K$。

K-Means 聚類與三簇質心分佈

肘部法則定位（拐角處即為最佳 K 值）

kmeans_elbow.py 實作代碼 
from sklearn.cluster import KMeans

wcss = []
for k in range(1, 11):
    kmeans = KMeans(n_clusters=k, init='k-means++', random_state=42)
    kmeans.fit(X)
    wcss.append(kmeans.inertia_) # inertia_ 即為 WCSS

思考與習題：

什麼是 K-Means++ 初始化策略？它與傳統的隨機隨機初始化相比，如何克服陷入局部極小值（Local Minima）的問題？

08

單純貝氏與 PR 曲線 (Precision-Recall Curve)

單純貝氏（Naive Bayes）基於機率經典貝氏定理，並做出關鍵的**「特徵條件獨立性假設」**（特徵之間互不干涉）：

$$P(Y \mid X) = \frac{P(Y) \prod_{d=1}^{D} P(X_d \mid Y)}{P(X)}$$

精確度-召回率（Precision-Recall）曲線：相較於 ROC 曲線對不平衡樣本的遲鈍，PR 曲線聚焦於正類。橫軸為召回率（Recall，全體真實正類中檢出多少），縱軸為精確度（Precision，預測為正類中真正準確的比例）。PR 曲線下面積稱為平均精準度（Average Precision, AP），值越高，代表面對少數派（如癌症、詐欺）的捕捉效能越頂尖。

不平衡數據集分類器的 Precision-Recall 權衡曲線

nb_pr_curve.py 實作代碼 
from sklearn.naive_bayes import GaussianNB
from sklearn.metrics import precision_recall_curve

nb = GaussianNB()
nb.fit(X_train, y_train)

y_scores = nb.predict_proba(X_test)[:, 1]
precision, recall, _ = precision_recall_curve(y_test, y_scores)

思考與習題：

單純貝氏算法中，如果測試集中某個特徵在某類別下的出現概率為 0（例如某個詞彙未在垃圾郵件中見過），為什麼會導致聯合預測機率直接歸 0？如何引入拉普拉斯平滑（Laplace Smoothing）來解圍？

梯度提升樹與訓練收斂線 (Loss Reduction)

梯度提升樹（Gradient Boosting Tree，如 XGBoost、LightGBM）採用加法模型（Additive Training）與前向分步算法，通過擬合上一步弱分類器的殘差或負梯度，逐步逼近最佳極限。

以 XGBoost 為例，其優化目標包含了二階泰勒展開式的損失與樹結構正則化懲罰項：

$$\mathcal{L}^{(t)} \approx \sum_{i=1}^{N} \left[ g_i f_t(x_i) + \frac{1}{2} h_i f_t^2(x_i) \right] + \gamma T + \frac{1}{2} \lambda \sum_{j=1}^{T} w_j^2$$

損失收斂線：觀察隨着弱分類器迭代樹（Boosting Iterations / Trees）的增加，訓練集與驗證集 Loss 的下降趨勢。若驗證集的 Loss 下降後又開始上揚，則代表在此迭代點開始發生**過擬合（Overfitting）**，應當在此點進行早停（Early Stopping）。

訓練集與驗證集 Boosting 迭代收斂曲線圖 (早停點提示)

xgboost_earlystop.py 實作代碼 
import xgboost as xgb

model = xgb.XGBClassifier(n_estimators=1000, max_depth=6)
# 在驗證集上若 10 次迭代 Loss 未下降則早停
model.fit(X_train, y_train, 
          eval_set=[(X_test, y_test)], 
          early_stopping_rounds=10, 
          verbose=False)

思考與習題：

與隨機森林可以同時並行計算每棵樹不同，為什麼標準梯度提升算法（Boosting）在多節點計算機群上很難直接對樹的訓練進行並行加速？它是如何克服此計算障礙的？

10

神經網路與學習曲線 (Learning Curve)

人工神經網路（Artificial Neural Networks, ANN）利用多層神經元與激活函數，理論上可逼近任意複雜度的連續非線性函數。模型的核心優化基礎是**反向傳播（Backpropagation）**與鏈式法則（Chain Rule）求導。

深度解碼學習曲線（Learning Curve）：比較訓練集表現（Training Score）與交叉驗證集表現（Cross-Validation Score）隨訓練樣本容量增加的趨勢，可以有效為模型進行診斷：

  • 高偏差 (欠擬合, Underfitting)：兩條曲線在偏低水準的評分處迅速貼靠。即便增加訓練樣本數，表現也無法顯著提升。
  • 高方差 (過擬合, Overfitting)：訓練集表現持續維持高位，但驗證集表現明顯落後，兩條線之間存有巨大鴻溝 (Gap)。此時引入更多樣本有助於逼合間隔。

樣本大小 vs 準確度之學習曲線診斷圖

mlp_learning_curve.py 實作代碼 
from sklearn.model_selection import learning_curve
from sklearn.neural_network import MLPClassifier

mlp = MLPClassifier(hidden_layer_sizes=(100, 50), max_iter=500)

# 自動獲取在不同訓練樣本長度下的準確度軌跡
train_sizes, train_scores, test_scores = learning_curve(
    mlp, X, y, cv=5, train_sizes=np.linspace(0.1, 1.0, 10)
)

思考與習題：

當神經網路過擬合時，在交叉驗證得分與訓練集得分的空間圖上，我們會觀察到什麼特徵？在結構中引入 L1/L2 正則化（Weight Decay）或 Dropout 為什麼能有效克服該現象？
